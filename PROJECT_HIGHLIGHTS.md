# 多麥健康烘焙電商專案技術亮點彙整報告

本報告旨在將「多麥健康烘焙」專案的優化成果，轉化為具備高度面試價值的技術亮點，幫助開發者在面試中展現其工程深度與產品思維。

## 一、 現代前端架構與工程化實踐

本專案採用 Vue 3 Composition API 搭配 Vite 進行開發，並嚴格遵循現代前端工程規範，確保專案具備高可維護性與可擴展性。

| 技術亮點 | 關鍵技術/實踐 | 面試價值展示 |
| :--- | :--- | :--- |
| **全域狀態管理** | Pinia (Vuex 5) | 熟練掌握**跨組件資料流**管理，將業務邏輯與 UI 徹底分離，符合**關注點分離 (Separation of Concerns)** 原則。 |
| **資料驅動設計** | Navbar, Footer, News 列表 | 實作**可配置化**組件，將靜態內容抽象為資料結構，為未來對接 CMS 或後端 API 奠定基礎。 |
| **表單驗證與控制** | VeeValidate + Yup | 掌握複雜表單的**前端防禦性編程**，確保資料輸入的嚴謹性與使用者體驗。 |
| **組件化與佈局** | Component-Based Architecture | 實作 `ProductCard`、`AppLayout` 等組件，展現對 **DRY (Don't Repeat Yourself)** 原則的堅持，提升程式碼複用性。 |

## 二、 效能優化與資源管理

本專案在效能方面進行了多層次的優化，以確保在不同網路環境下都能提供流暢的使用者體驗。

| 效能優化點 | 實作細節 | 效能貢獻 |
| :--- | :--- | :--- |
| **程式碼分割** | 路由懶載入 (Lazy Loading) | 透過 `import()` 實現路由級別的 Code Splitting，顯著減少首屏載入的 JS Bundle Size。 |
| **非同步組件載入** | `defineAsyncComponent` | 針對非核心組件（如服務條款彈窗）實現按需載入，進一步優化 LCP (Largest Contentful Paint)。 |
| **圖片載入優化** | Native Lazy Loading & Skeleton Screen | 使用 `loading="lazy"` 並搭配**骨架屏 (Skeleton)** 佔位符，減少 CLS (Cumulative Layout Shift) 並提升感知性能。 |
| **無限捲動** | Intersection Observer API | 應用於產品與新聞列表，以高效能方式實現無縫載入，避免傳統分頁的頁面跳轉。 |

## 三、 軟體品質與測試覆蓋

本專案具備完整的測試策略，展現對程式碼穩定性與品質的承諾。

| 測試亮點 | 實作細節 | 品質價值 |
| :--- | :--- | :--- |
| **單元測試 (Unit Testing)** | Vitest 搭配純函數邏輯 | 針對核心業務邏輯（如購物車計算、優惠券折扣）編寫測試，特別覆蓋**邊界條件**（如折扣後總價防護），證明程式碼的可靠性。 |
| **端到端測試 (E2E Testing)** | Playwright | 實作「完整購買流」測試，確保使用者從瀏覽到下單的**核心業務路徑**暢通無阻，降低回歸風險。 |
| **防禦性編程** | 優惠券折扣邏輯 | 確保折扣金額不會導致總價為負數，展現對**業務邏輯嚴謹性**的重視。 |

## 四、 使用者體驗 (UX) 與產品思維

本專案不僅關注技術，更融入了多項 UX 巧思，以提升產品的商業價值。

| UX 巧思 | 實作細節 | 商業價值 |
| :--- | :--- | :--- |
| **智慧免運提示** | 動態進度條與狀態管理 | 利用**遊戲化**思維，激勵使用者湊單達標，直接提升**平均客單價 (AOV)**。 |
| **即時互動回饋** | Toast 提示與流暢動畫 | 在加入購物車、篩選排序等操作後提供即時、非侵入式的回饋，提升使用者滿意度。 |
| **A11Y 基礎優化** | 語義化 HTML 與 `aria-label` | 確保網站具備基礎**可訪問性**，展現對使用者包容性與 Web 標準的尊重。 |
| **內容管理優化** | 分類篩選與分頁系統 | 解決大量內容的查找與瀏覽效率問題，提升資訊獲取速度。 |

---
**總結：**

本專案已從一個基礎的 Vue 練習作品，進化為一個具備**現代架構、多層次效能優化、完整測試覆蓋**，並融入**商業 UX 巧思**的專業級電商前端原型。在面試中，請務必將重點放在這些「為什麼」和「如何做」的工程決策上，而非僅僅是「做了什麼」。
