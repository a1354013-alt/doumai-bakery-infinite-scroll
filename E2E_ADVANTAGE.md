# 端到端測試 (E2E Testing) 面試價值分析 (E2E_ADVANTAGE.md)

## 核心價值：使用者旅程的穩定性

在面試中，E2E 測試展示了你對**產品整體穩定性**和**使用者核心旅程**的關注，證明你具備資深工程師的全局觀。

### 1. 確保核心業務路徑暢通 (Happy Path)
*   **說明**：我們使用 Playwright 實作了「從瀏覽商品 -> 加入購物車 -> 填寫結帳表單 -> 下單成功」的完整流程測試。
*   **面試說詞**：
    > 「我實作了 E2E 測試來確保應用程式的**核心業務路徑 (Critical User Journey)** 始終暢通。這能有效防止在部署新功能時，意外破壞了最關鍵的『結帳』流程，極大地降低了**回歸測試 (Regression Testing)** 的成本。」

### 2. 跨組件/跨頁面狀態同步驗證
*   **說明**：E2E 測試驗證了購物車數字在 Navbar、Cart Page、Checkout Page 之間的同步，以及下單後 Pinia 狀態的清空。
*   **面試說詞**：
    > 「E2E 測試不僅驗證單個組件，它還驗證了**跨組件的狀態同步**。例如，我們測試了下單成功後，Pinia Store 中的購物車狀態是否被正確清空，以及 Navbar 上的購物車數字是否歸零。這證明了系統的**資料一致性**。」

### 3. 響應式與瀏覽器兼容性
*   **說明**：Playwright 允許在不同尺寸和瀏覽器上運行測試。
*   **面試說詞**：
    > 「E2E 測試能模擬不同的裝置視圖（如手機），確保我們的**響應式設計**在不同螢幕尺寸下都能正常運作，特別是 Navbar 的漢堡選單等關鍵互動元件。」
